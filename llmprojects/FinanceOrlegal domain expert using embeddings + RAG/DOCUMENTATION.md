# Finance/Legal Domain Expert using Embeddings + RAG

## Project Overview
This project implements a finance and legal question-answering assistant using a retrieval-augmented generation (RAG) pipeline. It combines a LangChain-style conversational agent with a local knowledge base stored in `data/knowledge_base.txt`, a Chroma vector store, and embeddings generated by Ollama.

The app is designed as a small CLI program that reads user questions, optionally retrieves relevant knowledge base content via a tool, and then uses a language model to return answers.

## Project Structure & File Purposes

### Root files
- `main.py`
  - The main entry point for the application.
  - Implements a CLI loop that collects user prompts, invokes the graph-based workflow, prints either tool results or assistant answers, and logs a user rating for each turn.
  - Maintains conversation history and truncates it to the latest 8 messages.

- `graph.py`
  - Defines the agent workflow graph using `langgraph`.
  - Creates a state graph with two main nodes: `llm` and `tool`.
  - Routes execution from the LLM node to the tool node when a tool call is required; otherwise, it ends the flow.

- `requirements.txt`
  - Lists Python dependencies required to run the project.
  - Key dependencies include `langgraph`, `langchain-ollama`, `langchain-chroma`, `chromadb`, and text splitting utilities.

- `schemas.py`
  - Defines a Pydantic model `FinalAnswer` representing a structured answer format.
  - Includes `answer`, `confidence`, and optional `sources` fields.

- `state.py`
  - Defines the `State` typed dictionary used throughout the workflow.
  - Tracks `messages`, `tool_calls`, `tool_results`, `context`, `next_action`, and `turn_id`.

- `evaluation_log.json`
  - Stores logged evaluation metrics such as user ratings and retrieval success.
  - Updated by `utils/eval.py` during runtime.

- `summary.md`
  - A project summary artifact that may contain notes or findings.
  - Not required to run the app, but useful for documentation or review.

### Directories
- `.venv/`
  - Local Python virtual environment built for this project.
  - Contains the Python interpreter and installed dependencies.

- `data/`
  - Contains the knowledge base file used for retrieval.
  - `knowledge_base.txt` is loaded as the document source for vector search.

- `nodes/`
  - Contains the workflow nodes used by `graph.py`.
  - `llm_node.py`: Handles language model interaction and tool selection.
  - `tool_node.py`: Executes tool calls and appends tool results to the conversation.
  - `router.py`: Chooses the next workflow branch based on `state['next_action']`.

- `tools/`
  - Contains reusable tool definitions.
  - `retriever_tool.py`: Loads the knowledge base text, splits it into chunks, builds a Chroma vector store, and exposes `search_knowledge_base()` as a callable tool.

- `utils/`
  - Contains helper utilities.
  - `eval.py`: Provides logging functions for turn-level metrics and evaluation calculations.

## Important Components

### `nodes/llm_node.py`
- Uses `langchain_ollama.ChatOllama` with `model="llama3.1"` and `temperature=0`.
- Decides whether to call the knowledge base tool based on finance/legal keywords.
- When a tool result exists, it prompts the LLM to answer naturally using the retrieved data.
- Truncates the conversation to maintain a bounded LLM context window.

### `nodes/tool_node.py`
- Executes tool calls described in `state['tool_calls']`.
- Supports the `search_knowledge_base` tool.
- Logs retrieval success or errors using `utils.eval.log_metric()`.
- Adds tool results back into the message stream.

### `tools/retriever_tool.py`
- Loads `data/knowledge_base.txt` with `TextLoader`.
- Splits documents by character chunks for vector indexing.
- Builds a Chroma vector store with `OllamaEmbeddings(model="nomic-embed-text")`.
- Provides `search_knowledge_base(query)` to return the top 2 most relevant passages.

### `utils/eval.py`
- Provides `log_metric()` for appending metrics to `evaluation_log.json`.
- Includes `calculate_precision_at_k()` for evaluation calculations.

## How to Run the Project

### Prerequisites
- Python 3.13 is recommended, based on the `.venv` environment layout.
- `python3` or equivalent must be available on your system.
- `pip` for installing dependencies.
- Ollama runtime / server installed and configured if using `langchain_ollama`.
- The `nomic-embed-text` embedding model must be available to Ollama.
- No Docker setup is included; there is no `Dockerfile` or `docker-compose.yml` in this repository.

### Setup Steps
1. Clone or download the repository.
2. Open a terminal and change into the project directory:
   ```bash
   cd '/Users/techverito/llmprojects/FinanceOrlegal domain expert using embeddings + RAG'
   ```
3. Create and activate a virtual environment:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```
4. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
5. Verify the knowledge base file exists:
   ```bash
   ls data/knowledge_base.txt
   ```

### Running the App
Run the CLI program with:
```bash
python main.py
```

### What to Expect
- The program should print:
  ```text
  Finance/Legal Expert (Step 5 - Evaluation) - type 'quit' to exit
  ```
- Then enter a finance or legal question at the prompt.
- If the input contains finance/legal keywords, the agent will call the knowledge base retrieval tool.
- After the assistant response, you will be asked for a rating from `1` to `5`.
- Ratings and retrieval metrics are written to `evaluation_log.json`.

### Verifying the Project Works
- Confirm you can start the app with `python main.py`.
- Ask a sample question such as:
  ```text
  What does GDPR require for data protection?
  ```
- Confirm the app produces an answer or tool result.
- Check that `evaluation_log.json` is updated after entering a rating.

## Notes
- There is no `README.md`, `.env.example`, `Dockerfile`, or `docker-compose.yml` in this project.
- The repository relies on local Ollama integration and the `nomic-embed-text` embedding model.
- If Ollama is not configured, the app may fail when importing or using `langchain_ollama`.

## Project Structure & File Purposes

### Root files

- `main.py`
  - The main interactive entry point for the structured tool-calling agent.
  - Prompts the user, sends their input into the compiled `graph.app`, and prints either tool results or the assistant's final natural language answer.
  - Tracks state fields such as `messages`, `tool_calls`, `tool_results`, `next_action`, and `iterations`.

- `requirements.txt`
  - Lists Python dependencies required by the project:
    - `langgraph>=0.2.0`
    - `langchain-ollama>=0.1.0`
    - `langchain-core>=0.1.0`
    - `pydantic>=2.0.0`

- `graph.py`
  - Builds the LangGraph state machine for the agent.
  - Defines the graph flow: start at the LLM node, route to the tool node when `next_action` is `call_tool`, and return to the LLM after tool execution.
  - Uses `State`, `llm_node`, `tool_node`, and `router` to compose the agent pipeline.

- `state.py`
  - Declares the shared state schema used by the graph.
  - State contains:
    - `messages`: conversation history including human and AI messages
    - `tool_calls`: pending tool invocation requests
    - `tool_results`: outputs returned from executed tools
    - `next_action`: whether the graph should `call_tool` or `end`
    - `iterations`: counter to prevent runaway tool loops

- `summary.md`
  - A project artifact likely summarizing the project's goals or development progress.
  - Not required for runtime, but useful as a project overview note.

- `agent.log`
  - Log file generated by the logger utility.
  - Captures runtime events from the tool execution node.

### `tools/`

- `tools/calculator.py`
  - Implements a safe math expression evaluator used by the agent.
  - Supports arithmetic operations and functions from Python's `math` module.
  - Validates input to avoid unsafe characters and returns either a result string or an error string.

- `tools/weather.py`
  - Implements a mock weather tool.
  - Returns a randomly generated temperature and weather condition for a requested city.
  - Designed to simulate an external tool call rather than fetch real weather data.

- `tools/__init__.py`
  - Marks the `tools` package and enables package-level imports.

### `nodes/`

- `nodes/llm_node.py`
  - The LLM processing node for the agent.
  - Sends conversation history to `ChatOllama` and appends the model response.
  - Includes historic commented code showing iterative development through JSON tool call parsing and rule-based decision logic.
  - Ultimately acts as the agent's generative reasoning component.

- `nodes/tool_node.py`
  - The tool execution node.
  - Reads pending `tool_calls` from state, looks up tool handlers, executes them, and appends tool results as AI messages.
  - Uses a logger to write execution details to `agent.log`.
  - Clears `tool_calls` after execution and sets `next_action` to `end`.

- `nodes/router.py`
  - A simple router function that selects the next action based on `state["next_action"]`.
  - Returns either `call_tool` or `end`.

- `nodes/__init__.py`
  - Marks the `nodes` package for imports.

### `utils/`

- `utils/logger.py`
  - Provides a simple file logger configuration.
  - Writes debug-level logs to `agent.log` with timestamps and log levels.

## How to Run the Project

### Prerequisites

- Python 3.11 or later is recommended.
- `pip` package manager.
- Optional but recommended: use a virtual environment.
- No `Dockerfile` or `docker-compose.yml` is included in this repository.

### Setup Steps

1. Navigate to the project folder:
   ```bash
   cd '/Users/techverito/llmprojects/Module6 _Structured Tool-Calling Agent'
   ```

2. Create and activate a virtual environment:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. Install dependencies:
   ```bash
   python -m pip install --upgrade pip
   python -m pip install -r requirements.txt
   ```

4. Confirm the active interpreter is the virtual environment one:
   ```bash
   which python
   # Should point to /Users/techverito/llmprojects/Module6 _Structured Tool-Calling Agent/.venv/bin/python
   ```

### Running the Project

Run the interactive agent from the project root:

```bash
python main.py
```

This starts a CLI loop. Type questions or commands and press Enter. Enter `quit` to exit.

### Expected Runtime Behavior

- User input is appended to the conversation state.
- The LLM node in `nodes/llm_node.py` processes the query.
- If the LLM decides a tool is needed, it sets `state["tool_calls"]` and `next_action` to `call_tool`.
- `nodes/router.py` routes execution to `nodes/tool_node.py`.
- The tool node runs the requested tool(s), appends results to the conversation, and clears pending tool calls.
- Execution returns to the LLM node for follow-up reasoning or final response.

## How to Verify It’s Working

- Confirm the Python process starts without import errors when running `python main.py`.
- Enter a natural language prompt and verify the agent prints either an assistant response or a tool result.
- For weather-related queries, the agent should invoke the `get_weather` tool and print a result like `Weather in <city>: ...`.
- For calculation-related queries, the agent should invoke `calculate` and print `Result: ...` or an error message.
- Check `agent.log` for tool execution logs and debug entries.
